    A SMARTER WAY TO DUAL BOOT WINDOWS...¶

    https://www.youtube.com/watch?v=7JBFJuA5QsM

    This youtube video opens up possibility of keeping your existing windows installation and then build custom windows installs on top it.

    This method of creating Virtual Disks VHDX helps to keep multiple version of Windows on the same drive and same copies as well.

    Backing up of the Install is just copying out the .vhdx file.

    Another key advantage is that THis allows to port the Windows install between PC if such a thing is supported. Meaning that if you don't need too many drivers to be installed for the windows setup, you can port the install between PCs. This was impossible earlier but with this technique it will get enabled.

    In this demo, I show you a cool way to install multiple Windows operating systems on the same PC. No virtualization and No disk partitioning required.

    It's simple to add or remove an operating system at any time.

    The tools needed for the job are:

    1. diskpart This is the windows Partitioning utility available in Windows and Installation Media. We can use the Windows Disk Management to also create the VHDX but this one would be much universal.
    2. bcd-tools These are the collection of bcdlist and bcdedit That would allow to rename the Windows installs on the Virtual Disks
    3. msconfig This is used to reconfigure the boot priority and timing.

    Diskpart commands for formatting the disk to enable booting to Virtual Disks¶

    Let us now look at the process of partitioning and creating the VHDX files.

    You need to enter the Windows Setup and then press Shift + F10 to enter the Administrative command prompt. This would help to run various system commands.

    Diskpart commands for a UEFI BIOS Systems¶

    List all Disks available:

    list disk

    Select a Particular Disk where the Virtual Disks would be stored:

    select disk 0

    DANGER!! This command deletes all partitions on the selected Disk. Then creates a gpt partition table in the same disk.

    clean
convert gpt

    First Create the EFI boot partition:

    create partition efi size=200
format fs=fat32 label="System"

    In case you wish to also install Linux in a multi-boot configuration then set the size=2048. This would make the EFI Partition much bigger to host multiple Linux OS. Adjust the size and found optimal.

    Create the MSR needed for Windows installs:

    create partition msr size=128

    Finally create the Primary Partition where we would be storing our VHDX files. We would name this as Host-Drive to easily distinguish the drive. And finally also assign a drive letter so that we can install Windows.

    create partition primary
format fs=ntfs quick label="Host-Drive"
assign

    We can also limit the size of this Host-Drive with the following command.

    create partition primary size=512000

    This would create a 500GB partition.

    Note: All sizes are in Mega Bytes. The EFI partition is bigger to allow more OS to be installed.

    Diskpart commands for a Legacy BIOS System¶
    list disk
select disk 0
clean
create partition primary size=100
format fs=ntfs quick label="System"
assign
active
create partition primary
format fs=ntfs quick label="Host-Drive"
assign

    These are similar to the once for UEFI with some small changes.

    Diskpart commands to create a virtual disk¶

    create vdisk file=C:\filename.vhdx maximum=81920
select vdisk file=C:\filename.vhd
attach vdisk

    Change C:\ to whatever drive letter you have for your main hard disk

    Change 81920 to whatever size you want your disk to be (in MB) or just enter size to the nearest 1000, e.g 80000

    You can use an online calculator if you want to be exact.

    https://www.gbmb.org/gb-to-mb

    Using Diskpart To create a Expandable Virtual Disk¶

    create vdisk file=C:\filename.vhdx maximum=81920 type=expandable

    This would initially create a small file but then expand to a much bigger as the OS and files are installed over time.

    Diskpart commands to increase the size of a virtual disk¶

    Select the VHDX file that need extention.

    select vdisk file=D:\xxxxxxx.VHDX

    Where D:\xxxxx.VHDX is the path and filename of the virtual disk you want to increase.

    Expand the Disk size first to the Desired amount :

    expand vdisk maximum=102400

    Change 102400 to whatever size in MB you want to increase it to

    attach vdisk
list volume
select volume x:

    Change x: to whatever drive letter the attached virtual disk has got

    And finally extend the volume:

    extend

    Clean install Windows 11 on unsupported hardware - Alternative Method¶

    https://www.youtube.com/watch?v=WdxBbmWnMl0

    Another simple method for installing Windows 11 to unsupported hardware.

    This time we use the DISM tool to deploy the Windows 11 image to the hard disk thus bypassing the Windows 11 hardware check.

    Windows 11 CAN run on this PC !!!

    Microsoft Deployment Lab Script¶

    https://learn.microsoft.com/en-us/windows-hardware/manufacture/desktop/oem-deployment-of-windows-desktop-editions-sample-scripts?view=windows-11

    Look at the CreatePartitions-UEFI.txt Script:

    https://learn.microsoft.com/en-us/windows-hardware/manufacture/desktop/oem-deployment-of-windows-desktop-editions-sample-scripts?view=windows-11#createpartitions-uefitxt

    This would help to Partition the Disk into desired partitions using this command:

    dispart /s CreatePartitions-UEFI.txt

    Note: This might need to be customized for your desired PC or Disk Configuration.

    CreatePartitions-UEFI.txt
    rem == CreatePartitions-UEFI.txt ==
rem == These commands are used with DiskPart to
rem    create four partitions
rem    for a UEFI/GPT-based PC.
rem    Adjust the partition sizes to fill the drive
rem    as necessary. ==
select disk 0
clean
convert gpt
rem == 1. System partition =========================
create partition efi size=100
rem    ** NOTE: For Advanced Format 4Kn drives,
rem               change this value to size = 260 **
format quick fs=fat32 label="System"
assign letter="S"
rem == 2. Microsoft Reserved (MSR) partition =======
create partition msr size=16
rem == 3. Windows partition ========================
rem ==    a. Create the Windows partition ==========
create partition primary
rem ==    b. Create space for the recovery tools ===
rem       ** Update this size to match the size of
rem          the recovery tools (winre.wim)
rem          plus some free space.
shrink minimum=500
rem ==    c. Prepare the Windows partition =========
format quick fs=ntfs label="Windows"
assign letter="W"
rem === 4. Recovery partition ======================
create partition primary
format quick fs=ntfs label="Recovery"
assign letter="R"
set id="de94bba4-06d1-4d40-a16a-bfd50179d6ac"
gpt attributes=0x8000000000000001
list volume
exit

    Note: Make sure to copy this file to your windows installation USB. It would needed when the Windows Setup is running.

    Execute the Partition Script and DISM technique¶

    In the Windows 11 Installation press Shift + F10 to enter the Administrator Command Prompt.

    Make the partitions using the Script and Diskpart¶

    Change to the drive you have your Windows Installation media, and run Diskpart utility:

    d:

    diskpart \s "disprep.txt"

    Here the diskprep.txt contains the configuration done above.

    In case you wish to edit this file it would be made possible using notepad.exe.

    Type in command to edit the diskprep.txt file:

    notepad d:\diskprep.txt

    Make sure that your Path to the file is correct as per the Windows install USB disk drive letter.

    Use the DISM program to install the Correct version of the Windows 11 directly¶

    Use the command to list the OS Classification and Version available by using the command:

    dism /get-wiminfo /wimfile:"d:\sources\install.esd"

    Note: The drive letter for USB containing the Windows 11 Install source should be correct.

    Find out the Windows types using the Index: X number in the list that appears.

    Finally we are ready to begin the installation:

    dism /apply-image /imagefile:"d:\sources\install.esd" /index:6 /applydir:w:\

    This would install the index 6 which might be Windows 11 Pro into the w:\ drive mounted by our earlier step of partitioning using the script.

    Make the HDD Bootable for the currently installed Windows 11¶

    We select the w:\ drive to boot:

    bcdboot w:\windows /s s: /f UEFI

    This would make the current install bootable and store the UEFI boot in s:\ drive. This was created in the earlier partitioning step.

    1. Create a Local user in the Audit Mode¶

    Open a Administrator Command Prompt from the start menu.

    To create a user called madhav with their password as Password123:

    net user madhav Password123 /add

    For more details on the net user command refer to he Microsoft Documentation:

    https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/cc771865(v=ws.11)

    Next we set the Administrators privilege for madhav account:

    net localgroup administrators madhav /add

    For more details on the net localgroup command refer to the Microsoft Documentation:

    https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/cc725622(v=ws.11)

    You can now set the Administrator User password as AdminPass123=:

    net user Administrator AdminPass123=

    This would set the Password for the Administrator account.

    2. Login using the Local user in the Audit Mode to perform SysPrep¶

    After User creation and configuration is completed log out of the Administrator User and Login as madav or admin which ever you just created.

    Again, open a Administrator Command Prompt from the start menu.

    Change directory to the sysprep folder:

    cd c:\Windows\System32\Sysprep

    More details on the Sysprep tool from Microsoft Documentation:

    https://learn.microsoft.com/en-us/windows-hardware/manufacture/desktop/sysprep--generalize--a-windows-installation?view=windows-11

    Now there are two ways to Sysprep the images:

    1. Capture a generalized image to be deployable on multiple other machines
    sysprep /generalize /oobe /shutdown /unattend:d:\unattend.txt

    This would remove the uniqueness of the image and prepare it with shutting down.

    1. To only automate the OOBE on the machine (preferred):
    sysprep /oobe /reboot /unattend:d:\unattend.txt

    PC will then restart and complete the OOBE without any prompt from the user.
