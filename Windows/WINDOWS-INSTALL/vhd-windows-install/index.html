    This works for Windows 11, Windows 10 and Windows 8 into .vhdx Files without any Virtualization.

    But it does not work for Windows 7 as it does not support UEFI fully.

    https://www.youtube.com/watch?v=7JBFJuA5QsM

    This youtube video opens up possibility of keeping your existing windows installation and then build custom windows installs on top it.

    This method of creating Virtual Disks VHDX helps to keep multiple version of Windows on the same drive and same copies as well.

    Backing up of the Install is just copying out the .vhdx file.

    Another key advantage is that THis allows to port the Windows install between PC if such a thing is supported. Meaning that if you don't need too many drivers to be installed for the windows setup, you can port the install between PCs. This was impossible earlier but with this technique it will get enabled.

    In this demo, I show you a cool way to install multiple Windows operating systems on the same PC. No virtualization and No disk partitioning required.

    It's simple to add or remove an operating system at any time.

    The tools needed for the job are:

    1. diskpart This is the windows Partitioning utility available in Windows and Installation Media. We can use the Windows Disk Management to also create the VHDX but this one would be much universal.
    2. bcd-tools These are the collection of bcdlist and bcdedit That would allow to rename the Windows installs on the Virtual Disks
    3. msconfig This is used to reconfigure the boot priority and timing.

    To Create Virtual Disks and Enable Booting¶

    Let us now look at the process of partitioning and creating the VHDX files.

    You need to enter the Windows Setup and then press Shift + F10 to enter the Administrative command prompt. This would help to run various system commands.

    We would be using the diskpart utility that comes build it even with the installation media.

    This would allow us to partition the disk in the correct way.

    For more information refer to the documents on diskpart utility.

    We would later use the bcdboot utility or the EasyBCD utility in adjusting the boot loader manipulation.

    Here is how our disk partition would look like:

    # Partition Size (MB) Purpose
    1 EFI System 2048 Boot partition used in UEFI systems.
    2 MSR Partition 128 Secret Windows MSR Partition
    3 Host-Drive All space This is the location for VHDX file storage.
    4 ... ... Any other partitions that you need or Linux

    This below mention procedure needs to done only once to create one bootable VHDX installed Windows OS. For future ones refer to the Create additional VHDX guide.

    diskpart commands for a UEFI BIOS Systems to create Initial VHDX Storage¶

    We would be creating the required partition and configuring a disk to store all our VHDX file calling it VHDX Storage.

    1. Start diskpart¶

    To enter diskpart utility:

    diskpart

    2. List all Disks available¶

    list disk

    3. Select a Particular Disk where the Virtual Disks would be stored¶

    select disk 0

    How to chose the disk

    This might be daunting task if you are not aware of which disk to operate on. The first clues might lie in the Size of the Disk. The next on might be the Disk Names etc.

    4. Clear out the Partitions and prepare the Disk¶

    This step is needed since we are using entire disk for the VHDX storage.

    In-case its only a single partition on an already in use disk then DO Not follow through this step.

    clean
convert gpt

    DANGER!! This command deletes all partitions on the selected Disk. Then creates a gpt partition table in the same disk.

    5. Creating the EFI Partition¶

    Again this step is needed only if you wish to use your entire disk for the VHDX storage and do not have any existing EFI partitions.

    Create the EFI boot partition:

    create partition efi size=200
format fs=fat32 label="System"

    In case you wish to also install Linux in a multi-boot configuration or many Windows variants then set the size=2048.

    This would make the EFI Partition much bigger to host multiple Linux OS.

    Adjust the size and found optimal to use size=200.

    6. MSR Partition¶

    Again this one is a mandate in most cases. As its some mysterious thing need by Windows to identify the drives.

    Create the MSR needed for Windows installs:

    create partition msr size=128

    7. Primary Storage partition for VHDX storage¶

    Finally create the Primary Partition where we would be storing our VHDX files. We would name this as Host-Drive to easily distinguish the drive. And finally also assign a drive letter so that we can install Windows.

    create partition primary
format fs=ntfs quick label="Host-Drive"
assign
    Note that the last assign command would mount / attach the created VHDX Storage partition like into C: or D: or any such Drive Letter.

    We can also limit the size of this Host-Drive with the following command.

    create partition primary size=512000
format fs=ntfs quick label="Host-Drive"
assign

    This would create a 500GB partition.

    Note: All sizes are in Mega Bytes. The EFI partition is bigger to allow more OS to be installed.

    In order to ascertain which Drive Letter was assigned :

    list volume

    This would list our Host-Drive partition also.

    8. diskpart commands to create a virtual disk¶

    Note that we are assuming that C: is our Host-Drive partition. We found about out this earlier using the list vol command.

    create vdisk file=C:\filename.vhdx maximum=81920
select vdisk file=C:\filename.vhd
attach vdisk

    Change C:\ to whatever drive letter you have for your Host-Drive partition is assigned on the hard disk.

    Change 81920 to whatever size you want your disk to be (in MB) or just enter size to the nearest 1000, e.g 80000

    You can use an online calculator if you want to be exact.

    https://www.gbmb.org/gb-to-mb

    Using diskpart To create a Expandable Virtual Disk¶
    create vdisk file=C:\filename.vhdx maximum=81920 type=expandable
select vdisk file=C:\filename.vhd
attach vdisk

    This would initially create a small file but then expand to a much bigger as the OS and files are installed over time.

    9. Exit diskpart and Continue with Windows Install¶

    Now that the virtual disk is created you can exit diskpart.

    To exit diskpart utility on prompt:

    DISKPART> exit

    Finally continue the Windows 11 install.

    While selecting partition where to install choose the partition with VHDX Partition size of 80GB or what ever you set earlier. Since this is the VHDX file we mounted earlier. The windows will then be installed inside this file. Don't worry after booting you will also be able to see the Host-Drive without any issues.

    diskpart commands for the UEFI BIOS System to Create Next VHDX installs¶

    This is assuming that you have already completed the creation of the Host-Drive for VHDX Storage as explained the previous section.

    You need to enter the Windows Setup and then press Shift + F10 to enter the Administrative command prompt. This would help to run various system commands.

    1. Start diskpart¶

    This time the flow would be slightly different so don't overlook the steps.

    diskpart

    2. Check the Volumes and Find the Host-Drive¶

    list volume

    This would give an idea of the volumes that have been mounted and what Drive Letter has been assigned to them.

    In these you would find the volume named Host-Drive. This is where we are going to create our VHDX files. Make a note of the Drive Letter for the Host-Drive.

    3. Create the VHDX and Attach is it¶

    create vdisk file=C:\filename.vhdx maximum=81920 type=expandable
select vdisk file=C:\filename.vhd
attach vdisk

    This time we are going with an expandable one with familiar 80GB size.

    4. Exit diskpart and continue the install¶

    exit

    This would close the diskpart and bring you back to the terminal prompt. You can now close this Administrative Command prompt.

    Back in the Installation make sure select the 80GB Partition.

    5. Special Note on the Boot Order¶

    Whenever a new OS is install in this new VHDX we created here, its order becomes the first to start at boot.

    This means that when the install completes and you reboot into Boot Loader Screen aka Metro Boot Screen you will observe the the current OS becomes the new default OS that would start. Its a good idea to uniquely rename each of the OS as detailed later.

    Change the Windows Boot Name in the Metro Boot Loader Screen¶

    First boot into the desired OS and then use the following command in an Administrative Command Prompt:

    bcdedit /set {current} description "Windows 10 Demo"

    This would change the OS name for the current OS. You should see a Operation completed successfully as the output of this command.

    One needs to boot into each of the OS installed and then change the names.

    Or there is an easier way.

    EasyBCD Saves the day.

    Hiren’s BootCD PE

    This utility is available by default in all the editions of the Hiren’s BootCD PE. It might not be needed to install every time. You can just boot into the image and then do the manipulation based on the install order and the last installed instance.

    diskpart commands to increase the size of a virtual disk¶

    Open the diskpart tool in a Administrative Command Prompt.

    1. Select the VHDX file that need to expanded.

      select vdisk file=D:\xxxxxxx.VHDX

      Where D:\xxxxx.VHDX is the path and filename of the virtual disk you want to increase.

    2. Expand the Disk size first to the Desired amount :

      expand vdisk maximum=102400

      Change 102400 to whatever size in MB you want to increase it to

    3. Attach the Disk

      attach vdisk
list volume
select volume x:

      Change x: to whatever drive letter the attached virtual disk has got

    4. And finally extend the volume:

      extend

    5. Exit diskpart and reboot to make sure the changes take effect.

    Remove an OS that's not needed the VHDX way¶

    1. First delete the .VHDX file for the corresponding OS.

    2. Next open an Administrator Command Prompt and type:

      msconfig

      This can also be done in a Run Prompt by pressing Win+r Keys. 3. Then in the MS Config window in the Boot tab - select the OS entry to be removed and press Delete button.

    3. Then press Apply button to store this setting. And finally click on OK.

    4. You would be prompted to Restart the system, its recommended to do so for the changes to take effect.

    Backing up an OS installed in the VHDX Way¶

    The benefits of using a .VHDX file is that its a file on the disk that can be easily backed up and restored as needed.

    Just boot up into one of the other OS, different from the one you wish to backup.

    You can now just copy the file into a different disk for backup and restoration if needed in future.

    Optionally you can also use some utility such as zip, tar or 7z to archive the os to the different drive to store multiple versions and os for saving some space.

    Restoring an OS from Backup in the VHDX Way¶

    1. Boot into any of the OS other than the one to restore.

    2. Copy back the .VHDX file or extract its archive back to the Host-Disk Drive from where all OS boot. Make sure that it does not overwrite any of the existing images.

    3. The mount the .VHDX file into the OS using the Disk Management utility. In the Disk Management click on the Action Menu -> Attach VHD. Keep a note of the Drive letter its mounted to.

    4. Next open a Administrative Command Prompt and type

      bcdboot E:\Windows

      This would add the VHDX mounted to the E: drive letter as a boot entry.

    Easier way to Create the .VHDX files from Disk Management Utility¶

    1. Open the Disk Management Utility in one of the OSs you can boot into.
    2. Click on the Action Menu -> Create VHD This would open the Creation dialog.
    3. First need to specify the location and Filename .VHDX file
    4. Next we need to specify its size this time you can select the units for specifying the size.
    5. Next we need to choose the Virtual disk format this needs to be the VHDX as its the latest and better one.
    6. Next will be the type of Disk - New VHDX support dynamic disk and its recommended by default. This would create a disk that would increase in size as needed and not occupy the whole space at once.
    7. Click OK on the Creation dialog and then the Disk would be created and appear now in the Disk Management
    8. Next Right-click on the Disk and click Initialize. This would open the Initialize Disk dialog.
    9. Select the GPT partition type as its and click on OK. This would allow you to create partitions on the disk.
    10. Next inside the new initialized Disk area Right click and select New Simple Volume... This would open the New Volume creation Wizard. Go through the process and create the NTFS disk to access the whole of the space on the new disk created.
    11. This new partition would automatically be mounted and assigned a drive letter. Make a note of the letter and keep it handy for the next commands.
